3. システムアーキテクチャ

3.1 システム全体構成

[システム全体構成図]
ここにシステム全体の構成図を挿入

SES業務システムは、モノリシックアーキテクチャを採用し、論理的に分離された機能モジュールを備えた以下のレイヤー構造で構成される。

3.1.1 アーキテクチャ概要

  • プレゼンテーション層：ユーザーインターフェース
    • Web UI（レスポンシブデザイン）
    • モバイルアプリケーション
    • 管理者向けダッシュボード
  • API層：外部インターフェース
    • 統合RESTful APIエンドポイント
    • GraphQL API（※将来拡張予定）
  • アプリケーション層：業務ロジック
    • ドメイン駆動設計に基づく論理的に分離されたモジュール群
    • Spring MVCベースのコントローラ層
    • ワークフロー管理
  • データ層：データ管理
    • 統合リレーショナルデータベース（PostgreSQL）
    • キャッシュ（Redis）
    • オブジェクトストレージ（MinIO/S3互換）
  • インフラストラクチャ層：基盤
    • アプリケーションサーバー（Tomcat）
    • クラウドインフラ（AWS/Azure/GCP等）
    • CI/CDパイプライン
    • 監視・ロギング基盤

3.1.2 採用技術スタック

レイヤー 採用技術 選定理由
フロントエンド React, TypeScript, Redux コンポーネント指向のUI開発、型安全性、状態管理の容易さ
バックエンド Spring Boot 3.2.x, Java 21 エンタープライズでの実績、豊富なエコシステム、生産性
データベース PostgreSQL 17, Redis 高信頼性、統合データモデル、キャッシング
インフラ Application Server (Tomcat), Terraform 安定性、パフォーマンス、IaC（Infrastructure as Code）
CI/CD GitLab CI 自動化、継続的デリバリー
監視・ログ Prometheus, Grafana, ELK Stack 包括的な監視、可視化、ログ分析能力

3.2 モジュール構成

SES業務システムは、機能ごとに以下のモジュールに論理的に分割される。各モジュールはモノリシックアプリケーション内の論理的な境界を持つコンポーネントとして実装される。

3.2.1 業務モジュール

モジュール名 説明 主要機能
案件管理 クライアントからの案件情報を管理するモジュール 案件登録、案件検索、進捗管理、案件分析
技術者管理 自社エンジニアの情報を管理するモジュール 技術者登録、スキル管理、稼働状況管理、キャリア管理
マッチング 案件と技術者のマッチングを支援するモジュール マッチング検索、提案管理、AI推薦、適合度分析
契約管理 契約に関する情報を管理するモジュール 契約作成、電子署名連携、契約更新管理、契約書管理
勤怠・工数管理 技術者の勤怠と工数を管理するモジュール 勤怠入力、工数入力、承認フロー、実績集計
請求・支払管理 請求書発行と支払処理を管理するモジュール 請求書発行、入金管理、支払処理、税務管理
レポーティング 経営分析とレポート生成を行うモジュール ダッシュボード、KPI分析、予測分析、レポート出力

3.2.3 モジュール間の連携

モノリシックアーキテクチャ内のモジュール間連携は以下の方法で実現する：

  • 直接メソッド呼び出し: 依存性注入（DI）を用いたモジュール間の直接呼び出し
  • 内部イベント通知: Spring ApplicationEvent を利用したアプリケーション内イベント通知
  • 統合データモデル: 適切に設計されたデータモデルによるモジュール間の情報共有

3.2.4 共通コンポーネント

複数のモジュールで共有される共通コンポーネントを提供する：

  • 共通ユーティリティ: 日付操作、文字列操作、ファイル操作など
  • 例外処理: 一貫した例外階層とハンドリング
  • 監査ログ: 操作履歴の記録と管理
  • 設定管理: システム設定の集中管理
  • 国際化: 多言語対応の仕組み

3.3 サブシステム構成

SES業務システムは、モノリシックアーキテクチャを採用した以下の主要サブシステムから構成される：

3.3.1 フロントエンドサブシステム

  • Webアプリケーション: React + TypeScriptで実装されたSPA（Single Page Application）
  • モバイルアプリ: React Nativeで実装されたクロスプラットフォームアプリ
  • 管理者コンソール: システム管理者向けの専用インターフェース

3.3.2 バックエンドサブシステム

  • 統合Webアプリケーション: Spring Bootで実装された単一のモノリシックアプリケーション
  • 認証モジュール: アプリケーション内部の認証・認可コンポーネント
  • バッチ処理モジュール: Spring Batchベースの定期実行ジョブコンポーネント

3.3.3 データストアサブシステム

  • 統合データベース: 単一のPostgreSQL（マスター/スレーブ構成可）
  • キャッシュサーバー: Redis（アプリケーションキャッシュ）
  • オブジェクトストレージ: MinIO（S3互換ストレージ）

3.3.4 インテグレーションサブシステム

  • 外部連携コンポーネント: 外部システムとの連携モジュール
  • メール配信サービス: 通知メール送信コンポーネント
  • ファイル管理サービス: ドキュメント・添付ファイル管理コンポーネント

3.3.5 運用管理サブシステム

  • 監視サーバー: Prometheus/Grafanaによる監視・可視化
  • ログ管理: 集中ログ管理システム
  • CI/CDパイプライン: GitLab CIによる継続的インテグレーション・デリバリー
