ADM-004: ログ管理画面

画面概要

項目 内容
画面ID ADM-004
画面名 ログ管理画面
概要 システムログの閲覧・検索・エクスポートを行う画面
対象ユーザー システム管理者
前画面 CMN-002: ダッシュボード
次画面 ログ詳細モーダル（同画面内）

画面レイアウト

ログ管理画面の実際のモックアップは以下のとおりです。

画面動作仕様

ログ表示機能

  1. 画面初期表示時は、操作ログタブが選択され、直近24時間のログが表示される
  2. タブを切り替えると、対応するログ種別に表示が切り替わる
  3. ログ種別ごとに表示項目が一部異なる
    • 操作ログ：ユーザーID、操作対象、操作内容を重視
    • システムログ：カテゴリ、メッセージを重視
    • エラーログ：エラーレベル、メッセージを重視
    • セキュリティログ：ユーザーID、IPアドレス、アクション種別を重視
  4. 各カラムヘッダーをクリックすることで、そのカラムでソートができる

ログ検索機能

  1. 検索条件を入力し、検索ボタンをクリックする
  2. 入力された条件に合致するログを検索し、一覧に表示する
  3. 該当するログがない場合は「該当するログがありません」というメッセージを表示する
  4. クリアボタンをクリックすると、すべての検索条件をクリアする
  5. 検索条件はログ種別によって一部が有効/無効となる（例：ユーザーIDはシステムログでは無効）

ログ詳細表示機能

  1. 一覧の詳細表示リンクをクリックすると、ログ詳細モーダルが表示される
  2. ログの種類によって表示される詳細情報の内容が異なる
  3. エラーログの場合はスタックトレースが表示される
  4. 操作ログの場合は操作パラメータの詳細が表示される
  5. 関連ログがある場合は、リンクとして表示され、クリックすると該当ログの詳細が表示される

ログエクスポート機能

  1. エクスポートボタンをクリックする
  2. 現在の検索条件に合致するログをCSVファイルまたはJSONファイルにエクスポートする
  3. 大量データの場合は処理中にプログレスバーを表示する
  4. ファイルのダウンロードが開始される

ログ種別と記録内容

操作ログ

  • ユーザーがシステム上で行った操作の記録
  • 記録内容：日時、ユーザーID、IPアドレス、操作種別、操作対象、パラメータ、結果
  • 主な操作種別：登録、更新、削除、参照、承認、否認、ダウンロード、アップロード

システムログ

  • システム内部の処理状況や重要なイベントの記録
  • 記録内容：日時、ログレベル、カテゴリ、メッセージ、コンポーネント
  • 主なカテゴリ：アプリケーション、データベース、バッチ処理、外部連携

エラーログ

  • システム内で発生したエラーや例外の記録
  • 記録内容：日時、エラーレベル、エラーコード、エラーメッセージ、スタックトレース、コンテキスト情報
  • エラーレベル：WARN（警告）、ERROR（エラー）、FATAL（致命的）

セキュリティログ

  • 認証・認可に関する操作や不正アクセスの試みなどセキュリティ関連の記録
  • 記録内容：日時、ユーザーID、IPアドレス、アクション種別、結果、詳細情報
  • 主なアクション：ログイン、ログアウト、認証失敗、権限変更、パスワード変更、多要素認証

特記事項

  • ログ管理画面はシステム管理者ロールを持つユーザーのみアクセス可能
  • ログの保持期間は以下の通り設定される
    • 操作ログ：3年間
    • システムログ：1年間
    • エラーログ：2年間
    • セキュリティログ：5年間
  • 保持期間を過ぎたログは自動的にアーカイブされる
  • アーカイブされたログは別途アーカイブ閲覧ツールで参照可能
  • パフォーマンス上の理由から、一度に表示するログは最大1000件まで
  • 個人情報や機密情報はログ内でマスキングされる
  • 緊急時のログ収集機能として、指定条件のログを即時ダウンロードできる機能を提供
