画面動作仕様

バッチ一覧表示機能

  1. 画面初期表示時は、定期バッチタブが選択された状態で表示される
  2. タブを切り替えると、定期バッチ/オンデマンドバッチ/実行履歴に表示が切り替わる
  3. カテゴリフィルタで特定カテゴリのバッチのみを表示できる
  4. 検索ボックスに文字を入力すると、バッチID・名称でフィルタリングされる
  5. 各カラムヘッダーをクリックすることで、そのカラムでソートができる

バッチ手動実行機能

  1. 一覧の操作カラムから実行ボタンをクリックする
  2. パラメータ設定モーダルが表示される
  3. 必要なパラメータを入力し、実行ボタンをクリックする
  4. バッチ処理が非同期で実行される
  5. 実行開始時に通知メッセージが表示される
  6. 実行履歴タブに遷移して、最新の実行状況を確認できる

スケジュール設定機能

  1. 一覧の操作カラムから設定ボタンをクリックする
  2. スケジュール設定モーダルが表示される
  3. ウィザード方式またはCron式方式でスケジュールを設定する
  4. 設定に基づいて次回実行予定日時が表示される
  5. 保存ボタンをクリックすると設定が保存される
  6. スケジュール有効化チェックボックスで実行の有効/無効が切り替えられる

パラメータ設定機能

  1. 一覧の操作カラムから設定ボタンをクリックし、パラメータタブを選択する
  2. パラメータ設定モーダルが表示される
  3. バッチ処理ごとに定義されたパラメータを設定する
  4. リトライ設定、タイムアウト、通知設定を行う
  5. 保存ボタンをクリックすると設定が保存される

実行履歴表示機能

  1. 実行履歴タブをクリックする
  2. 期間やバッチ選択などの条件で履歴を検索できる
  3. 結果フィルタで成功/失敗/警告のステータスで絞り込みが可能
  4. 詳細ボタンをクリックすると、実行詳細モーダルが表示される
  5. 実行詳細ではログや結果サマリー、エラー詳細を確認できる
  6. ログや結果ファイルをダウンロードできる
  7. 必要に応じて同じパラメータで再実行することも可能

バッチ処理カテゴリと主要バッチ

データ集計バッチ

  • 日次売上集計：前日の売上データを集計
  • 月次売上集計：月次の売上データを集計
  • 稼働率集計：技術者の稼働率を集計
  • 粗利集計：案件ごとの粗利を集計

請求・支払バッチ

  • 請求書生成：月次の請求書データを生成
  • 支払データ生成：月次の支払データを生成
  • 入金消込：入金データと請求データの消込処理
  • 未入金通知：支払期限を過ぎた未入金の通知

通知バッチ

  • 勤怠未提出通知：勤怠未提出者への通知
  • 契約更新通知：契約更新予定の通知
  • 契約終了通知：契約終了予定の通知
  • アラート通知：KPI閾値超過時のアラート通知

データメンテナンスバッチ

  • データアーカイブ：古いデータのアーカイブ処理
  • ログクリーンアップ：古いログの削除
  • テンポラリファイルクリーンアップ：一時ファイルの削除
  • バックアップ：データのバックアップ処理

インポート/エクスポートバッチ

  • マスタデータインポート：外部システムからのマスタデータ取込
  • 取引先データ連携：会計システムへのデータ連携
  • 社員データ連携：人事システムとのデータ連携
  • データエクスポート：各種データの外部システム向けエクスポート

特記事項

  • バッチ管理画面はシステム管理者ロールを持つユーザーのみアクセス可能
  • バッチ処理の実行はシステムリソースを消費するため、同時実行数に制限がある（デフォルト：5）
  • 長時間実行するバッチ処理は、タイムアウト設定を適切に行う必要がある
  • 定期バッチのスケジュール設定は、システム負荷の分散を考慮して行う
  • 緊急時にバッチを停止できる「緊急停止」機能を提供する
  • バッチ実行履歴は3ヶ月間保持され、それ以降は自動的にアーカイブされる
  • バッチ処理のログは詳細モードと通常モードを切り替え可能
  • 重要なバッチ処理の失敗時は、管理者へのメール通知と連携して運用する
