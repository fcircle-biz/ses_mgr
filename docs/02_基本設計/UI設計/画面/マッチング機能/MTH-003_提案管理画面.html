画面動作仕様

  1. 画面初期表示時は、以下のように動作する：
    • 新規提案：空の提案フォームを表示
    • マッチング詳細からの遷移：マッチング情報を基に案件・技術者情報が自動入力される
    • 提案編集：既存の提案情報が入力された状態で表示
  2. 「提案単価」を入力すると、技術者の単価情報から自動的に「想定粗利率」が計算される
  3. 「案件ID」「案件名」をクリックすると、該当の案件詳細画面（PRJ-003）に遷移する
  4. 「技術者ID」「技術者名」をクリックすると、該当の技術者詳細画面（ENG-003）に遷移する
  5. 「履歴追加」ボタンをクリックすると、履歴追加用のモーダルダイアログが表示される
  6. 「保存ボタン」をクリックすると、入力された提案情報が保存され、同じ画面に留まる
  7. 「提案書作成ボタン」をクリックすると、提案書作成画面（MTH-004）に遷移し、現在の提案情報が引き継がれる
  8. 「提案一覧ボタン」をクリックすると、提案一覧画面に遷移する
  9. 「キャンセルボタン」をクリックすると、変更がキャンセルされ、マッチング詳細画面（MTH-002）に戻る
  10. 提案ステータスを変更すると、自動的に履歴が追加される

注意： 提案情報は案件・技術者の稼働状況など、関連情報と整合性を保つ必要があります。

  • 技術者の稼働状況が「稼働中」「予約済み」の場合、提案ステータスが「採用」に変更できないようにする制約があります
  • 案件ステータスが「成約済み」の場合、新規提案ができないようにする制約があります
  • 提案ステータスが「採用」に変更された場合、技術者の稼働状況を「予約済み」に自動変更します
