CNT-002: 契約作成画面

画面概要

項目 内容
画面ID CNT-002
画面名 契約作成
概要 新規契約情報を作成する画面
対象ユーザー 営業担当者
前画面 CNT-001: 契約一覧
次画面 CNT-003: 契約詳細（作成成功時）、CNT-004: 契約書プレビュー（プレビュー時）、CNT-001: 契約一覧（キャンセル時）

画面動作仕様

  1. 必須項目が入力されているかをバリデーションチェックする
    • 未入力の必須項目がある場合はエラーメッセージを表示
    • 入力形式や値の範囲が不正な場合はエラーメッセージを表示
  2. 契約テンプレートを選択すると、選択したテンプレートに応じた入力項目が表示される
  3. 自動更新チェックボックスがオンの場合、契約終了日は任意入力となり、更新通知日数が必須となる
  4. 支払条件で「締め日払い」を選択した場合、支払日の入力が必須となる
  5. 「保存」ボタンをクリックした場合
    • 入力内容のバリデーションを行い、問題がなければ契約情報を下書き保存
    • 保存成功時は契約詳細画面（CNT-003）に遷移する
    • 保存失敗時はエラーメッセージを表示し、入力内容を保持したままの状態で再表示する
  6. 「プレビュー」ボタンをクリックした場合
    • 入力内容のバリデーションを行い、問題がなければ契約書プレビュー画面（CNT-004）に遷移する
    • バリデーションエラーがある場合は、エラーメッセージを表示
  7. 「承認依頼」ボタンをクリックした場合
    • 入力内容のバリデーションを行い、問題がなければ契約情報を保存し、承認者に承認依頼を送信
    • 承認依頼成功時は契約詳細画面（CNT-003）に遷移し、承認待ちステータスが表示される
    • 承認依頼失敗時はエラーメッセージを表示
  8. 「キャンセル」ボタンをクリックした場合
    • 入力内容を破棄して契約一覧画面（CNT-001）に戻る
    • 入力内容がある場合は確認ダイアログを表示し、ユーザーの意思を確認する
