勤怠管理画面

画面概要

画面ID: TMS-003
画面名: 勤怠管理画面
対象ユーザー: プロジェクトマネージャー、管理者
前画面: ダッシュボード、メニュー画面
次画面: 勤怠詳細画面、勤怠承認画面

勤怠管理画面は、管理者やプロジェクトマネージャーが部下や部門メンバーの勤怠状況を一覧で確認し、管理するための画面です。 月次の勤怠データのステータス確認、承認待ちデータの処理、異常値の検出など、勤怠データの管理業務を効率的に行うことができます。 また、個人別の勤怠詳細データの閲覧や、勤怠データに関する各種指標の確認も可能です。

画面レイアウト

[勤怠管理画面のワイヤーフレーム表示位置]

検索・フィルタリング条件部分、勤怠サマリー、勤怠一覧テーブル、一括操作ボタン類から構成される画面

画面は主に以下の3つのエリアで構成されます：

  1. 検索・フィルタリング部分：年月選択、部門・プロジェクト選択、ステータスフィルターなど
  2. 勤怠サマリー部分：該当月の勤怠集計情報（人数、提出率、承認率など）
  3. 勤怠一覧テーブル：技術者別の勤怠データ一覧（ステータス、勤務日数、稼働時間などを表示）

画面動作仕様

1. 初期表示

  • 初期表示時は当月の勤怠データを表示
  • ログインユーザーの権限に応じて閲覧可能な技術者の勤怠データのみを表示
  • 管理者：全社員データを閲覧可能
  • 部門マネージャー：配下部門の社員データを閲覧可能
  • プロジェクトマネージャー：担当プロジェクトのメンバーデータを閲覧可能
  • 勤怠データはステータス順にソート（未提出 → 提出済 → 差戻 → 承認済 → 確定）

2. 検索・フィルタリング

  • 年月を変更すると、該当月の勤怠データを再読込
  • 部門またはプロジェクトを選択すると、該当する技術者の勤怠データにフィルタリング
  • ステータスフィルタを選択すると、該当するステータスの勤怠データにフィルタリング
  • キーワード検索は技術者名、技術者ID、案件名などに対して部分一致検索
  • 「異常値のみ表示」にチェックを入れると、以下の条件に該当するデータのみを表示
    • 月の半分以上が未入力の勤怠データ
    • 法定超過残業（45時間/月超）がある勤怠データ
    • 深夜勤務や休日勤務が規定値を超える勤怠データ
    • 未提出または差戻状態で締め日を過ぎている勤怠データ

3. 勤怠データ操作

  • 技術者氏名をクリックすると、その技術者の勤怠詳細画面に遷移
  • 操作ボタン群から「詳細」をクリックすると、勤怠データの詳細画面に遷移
  • 操作ボタン群から「承認」をクリックすると、承認確認ダイアログを表示
  • 操作ボタン群から「差戻」をクリックすると、差戻理由入力ダイアログを表示
  • 操作ボタン群から「編集」をクリックすると、管理者権限による勤怠データ編集画面に遷移

4. 一括操作

  • 一括操作を行う技術者を選択チェックボックスで選択
  • 「一括承認」ボタンをクリックすると、選択した提出済みの勤怠データを一括で承認処理
  • 「一括差戻」ボタンをクリックすると、差戻理由入力ダイアログを表示し、入力後に一括で差戻処理
  • 「未提出リマインド送信」ボタンをクリックすると、未提出者にリマインドメールを送信
  • 「勤怠データエクスポート」ボタンをクリックすると、表示中のデータをCSV/Excel形式でダウンロード
  • 「帳票出力」ボタンをクリックすると、勤怠帳票の出力フォーマット選択ダイアログを表示

5. テーブル操作

  • テーブルヘッダーをクリックすると、該当列でソート（昇順/降順切替）
  • 表示件数を選択して、1ページあたりの表示数を変更可能
  • ページネーションにより、大量データも管理可能

6. 異常値の表示

  • 残業時間が45時間を超える場合、該当セルを赤色で強調表示
  • 未入力日数が多い場合、警告アイコンを表示
  • 締め日を過ぎても未提出・未承認の場合、警告アイコンを表示

特記事項・注意点

  • 勤怠データの一括承認は、内容を十分確認した上で行うこと
  • 承認権限は組織階層やプロジェクト設定に基づいて管理され、承認可能な対象のみ操作ボタンが有効化される
  • 月次締め処理後は、原則として勤怠データの修正は不可（特別な権限が必要）
  • 残業時間や深夜勤務に関する法定基準値を超えるデータは常に警告表示し、適切な対応を促す
  • 個人情報保護の観点から、エクスポートされるデータへのアクセス権限を適切に管理する必要がある
  • 大量データ処理時のパフォーマンスを考慮し、初期表示データ量の制限や非同期読み込みを実装
