工数入力画面

画面概要

画面ID: TMS-002
画面名: 工数入力画面
対象ユーザー: 技術者
前画面: 勤怠入力画面（TMS-001）、ダッシュボード
次画面: 勤怠入力画面（TMS-001）、工数集計画面（TMS-006）

工数入力画面は、技術者が日々の作業内容と作業時間（工数）を案件・タスク別に記録するための画面です。 勤怠入力で登録した総労働時間を、異なる業務やプロジェクトにどれだけ配分したかを入力します。 入力された工数データは、案件別の稼働状況管理や請求処理の基礎データとなります。

画面レイアウト

[工数入力画面のワイヤーフレーム表示位置]

日付選択部分、勤怠情報サマリー、工数入力テーブル、工数合計表示、ボタン類から構成される画面

画面は主に以下の4つのエリアで構成されます：

  1. ヘッダー部分：日付選択、前日/翌日移動ボタン
  2. 勤怠情報表示部分：選択日の勤怠情報（勤務状態、実働時間など）のサマリー表示
  3. 工数入力テーブル：案件/タスク別の工数を入力するテーブル（行追加可能）
  4. フッター部分：工数合計、実働時間との差分表示、操作ボタン

画面動作仕様

1. 初期表示

  • 初期表示時は当日の日付が選択された状態で表示
  • 勤怠データが入力済みの場合、勤怠情報サマリー部分に情報を表示
  • 勤怠データが未入力の場合、警告メッセージを表示
  • 工数データが既に入力済みの場合は、そのデータをテーブルに表示
  • 工数データが未入力の場合は、空の工数入力行を1行表示

2. 日付変更時

  • 日付選択、または前日/翌日ボタンで日付を変更すると、選択された日付の勤怠情報と工数データを再読込
  • 未保存の工数データがある場合は、確認ダイアログを表示
  • 選択された日付が未来日付の場合、工数入力を無効化

3. 工数入力時

  • 案件を選択すると、その案件に関連する既定のタスク分類や請求可能フラグの初期値を自動設定
  • 工数を入力すると、リアルタイムで合計工数と実働時間との差分を再計算
  • 工数合計が実働時間と異なる場合、差分を赤字で表示し警告
  • 工数合計が実働時間を超える場合、超過分を強調表示

4. 工数テンプレート操作

  • テンプレート保存ボタンをクリックすると、現在の工数入力内容に名前を付けて保存するダイアログを表示
  • テンプレート読込ボタンをクリックすると、保存済みテンプレート一覧を表示し、選択したテンプレートを読み込む
  • 前日コピーボタンをクリックすると、前日の工数データを現在の画面にコピー（案件とタスクのみ、工数値は空欄）

5. 保存時

  • 必須項目が未入力の場合、エラーメッセージを表示して保存を中止
  • 工数合計と実働時間が異なる場合、警告メッセージを表示するが、保存は可能
  • 入力内容に問題がなければ、データを保存し、成功メッセージを表示
  • 保存後、画面をリフレッシュして最新データを表示

6. 週表示モード

  • 週表示切替ボタンをクリックすると、現在の日付を含む週（月～金または月～日）の工数を一画面で入力可能な表示に切り替え
  • 週表示モードでは、日ごとの列と、案件・タスクごとの行のマトリクス形式で工数を入力
  • 各日の合計工数と実働時間の整合性を常に確認可能

特記事項・注意点

  • 勤怠情報の実働時間と工数合計の整合性を取ることが重要
  • 承認済みの工数データは原則として編集不可（修正が必要な場合は承認者または管理者による操作が必要）
  • 頻繁に入力する内容はテンプレートとして保存し、効率化を図る
  • 休暇・休日の場合も工数入力は可能（休暇中の自己研鑽等の記録用）
  • アサイン中の案件のみが選択可能（終了案件は選択不可）
  • 請求可能フラグは請求・支払処理に直接影響するため、正確に設定が必要
  • 工数入力は月次勤怠申請の前提条件となるため、月末までに漏れなく入力する必要がある
