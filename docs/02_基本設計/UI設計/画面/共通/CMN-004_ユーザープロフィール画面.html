画面動作仕様

  1. ページ初期表示時
    • ログインユーザーのプロフィール情報を取得して表示する
    • パスワード変更セクションは初期状態で非表示
  2. プロフィール画像の更新
    • プロフィール画像をクリックすると画像アップロードダイアログが表示される
    • 新しい画像をアップロードするとプレビューが表示される
  3. パスワード変更
    • 「パスワード変更」ボタンをクリックすると、パスワード変更セクションが表示される
    • 現在のパスワード、新しいパスワード、確認用パスワードを入力する
    • 新しいパスワードと確認用パスワードが一致しない場合はエラーメッセージを表示
    • 現在のパスワードが誤っている場合はエラーメッセージを表示
  4. 多要素認証設定
    • チェックボックスで多要素認証を有効/無効にできる
    • 初めて有効化する場合は、QRコードとリカバリーコードが表示される
    • 無効化する場合は、確認ダイアログが表示される
  5. 保存ボタンクリック時
    • 入力値のバリデーションを行う
    • 必須項目が未入力の場合はエラーメッセージを表示
    • 入力形式が不正な場合はエラーメッセージを表示
    • すべての検証が成功した場合、プロフィール情報を更新する
    • 更新完了後、完了メッセージを表示する
  6. キャンセルボタンクリック時
    • 変更内容を破棄し、元の値に戻す
    • 変更内容がある場合は確認ダイアログを表示する

特記事項

  • プロフィール情報の一部は組織管理者によって管理される場合があり、その場合はユーザーによる変更ができない項目が存在する
  • 多要素認証を初めて有効化した際のリカバリーコードは必ずユーザーに保存を促す通知を表示する
  • パスワード変更が成功した場合、セキュリティ強化のため再ログインを要求する
  • メールアドレスを変更した場合、新しいメールアドレスに確認メールを送信し検証するプロセスを実装する
