CMN-004: ユーザープロフィール画面

画面概要

項目 内容
画面ID CMN-004
画面名 ユーザープロフィール画面
概要 ユーザー自身のプロフィール情報を参照・更新するための画面
対象ユーザー 認証済みのすべてのユーザー
前画面 CMN-002: ダッシュボード、CMN-003: メニュー
次画面 各メニュー画面

画面レイアウト

画面構成要素

1. プロフィール情報サイドバー

コンポーネント 説明 機能・振る舞い
プロフィール画像 ユーザーのプロフィール画像
  • クリックするとファイル選択ダイアログを表示
  • 画像をアップロードするとプレビュー表示
ユーザー情報サマリー ユーザー名、ID、部署、役職の表示 現在のユーザー情報を概略表示
連絡先情報 メールアドレス、電話番号の表示 現在設定されている連絡先情報を表示
システム設定情報 言語、タイムゾーン、多要素認証の状態表示 現在設定されているシステム設定を表示

2. 個人情報セクション

コンポーネント 説明 機能・振る舞い
氏名フィールド ユーザーの氏名入力欄 必須項目、最大50文字
氏名（カナ）フィールド ユーザーの氏名（カナ）入力欄 必須項目、全角カナ、最大50文字
部署フィールド 所属部署名入力欄 任意項目、最大50文字
役職フィールド ユーザーの役職入力欄 任意項目、最大30文字
メールアドレスフィールド 連絡先メールアドレス入力欄 必須項目、メールアドレス形式
電話番号フィールド 連絡先電話番号入力欄 任意項目、数字とハイフン、最大15文字

3. システム設定セクション

コンポーネント 説明 機能・振る舞い
言語設定ドロップダウン インターフェース言語選択 日本語/英語から選択
タイムゾーンドロップダウン ユーザーのタイムゾーン設定 利用可能なタイムゾーンから選択

4. セキュリティ設定セクション

コンポーネント 説明 機能・振る舞い
パスワード変更ボタン パスワード変更セクションの表示切替 クリックでパスワード変更フォームの表示/非表示を切替
現在のパスワードフィールド 現在のパスワード入力欄 パスワード変更時の確認用、表示/非表示切替可能
新しいパスワードフィールド 新しいパスワード入力欄 半角英数記号、12〜30文字、表示/非表示切替可能
新しいパスワード（確認）フィールド 新しいパスワードの確認入力欄 新しいパスワードと一致する必要あり、表示/非表示切替可能
多要素認証トグル 多要素認証の有効/無効切替 チェックでオン/オフ、初回有効化時は設定が必要
多要素認証セットアップ 初回有効化時のセットアップ表示 QRコード、設定キー、リカバリーコードの表示

5. アクションボタン

コンポーネント 説明 機能・振る舞い
保存ボタン 変更内容を保存するボタン クリックで変更内容をサーバに送信し保存
キャンセルボタン 変更内容を破棄するボタン クリックで変更をキャンセルし、元の値に戻す

画面動作仕様

1. ページ初期表示時の動作

処理 内容
プロフィール情報取得 ログインユーザーのプロフィール情報をAPIから取得して表示する
パスワード変更セクション 初期状態では非表示
多要素認証状態 現在の設定状態に応じてトグルの状態を設定

2. プロフィール画像の更新

ユーザー操作 システム応答
プロフィール画像クリック ファイル選択ダイアログを表示
画像ファイル選択 選択された画像をプレビュー表示
保存ボタンクリック 選択された画像をアップロードして保存

3. パスワード変更

ユーザー操作 システム応答
パスワード変更ボタンクリック パスワード変更セクションの表示/非表示を切替
表示切替アイコンクリック パスワードの表示/非表示を切替
必要情報入力後、保存ボタンクリック 入力内容を検証後、パスワードを更新

4. 多要素認証設定

ユーザー操作 システム応答
多要素認証トグルを有効に切替（初回） 多要素認証セットアップセクションを表示
QRコードスキャンまたはキー入力 認証アプリで設定完了
確認コード入力 コード検証後、多要素認証を有効化
多要素認証トグルを無効に切替 確認ダイアログを表示し、承認後に無効化

5. フォーム検証と保存

検証項目 検証内容
必須項目チェック 氏名、氏名（カナ）、メールアドレス、言語設定、タイムゾーンが必須
メールアドレス形式チェック メールアドレスが正しい形式か検証
文字数制限チェック 各フィールドの文字数が制限内か検証
カナ文字チェック 氏名（カナ）が全角カナのみか検証
パスワード一致チェック 新パスワードと確認用パスワードが一致するか検証
パスワード複雑性チェック パスワードが英数記号を含む12文字以上か検証

注意： 本画面はユーザー自身の情報を管理するためのものであり、個人情報保護の観点から適切なセキュリティ対策が必要です。パスワード変更や多要素認証設定など、重要な設定変更を行った場合は、登録されたメールアドレスに通知メールを送信します。また、組織によっては一部の情報が管理者によって制限されている場合があり、その場合は該当項目が編集不可として表示されます。メールアドレス変更時には、新しいアドレスに確認メールを送信し、リンククリックによる確認が完了するまでは旧アドレスが使用されます。多要素認証を初めて有効化する際は、リカバリーコードを安全な場所に保存するよう促す通知を表示します。
