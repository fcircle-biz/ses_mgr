CMN-005: 通知一覧画面

画面概要

項目 内容
画面ID CMN-005
画面名 通知一覧画面
概要 ユーザー宛てのシステム通知・アラートを一覧表示し、既読/未読管理を行う画面
対象ユーザー 認証済みのすべてのユーザー
前画面 CMN-002: ダッシュボード、CMN-003: メニュー
次画面 通知内容に応じた関連画面、または各メニュー画面

画面レイアウト

画面構成要素

1. ヘッダー部分

コンポーネント 説明 機能・振る舞い
画面タイトル 「通知一覧」のタイトル表示 画面の目的を示す
通知設定ボタン 通知設定画面へのリンクボタン クリックで通知設定モーダルを表示

2. 検索・フィルタリングセクション

コンポーネント 説明 機能・振る舞い
検索ボックス 通知内容を検索するためのテキスト入力欄 入力すると通知リストをリアルタイムでフィルタリング
期間選択ドロップダウン 通知の期間範囲を指定するセレクトボックス 選択した期間内の通知のみを表示
すべて既読ボタン 表示されている未読通知をすべて既読にするボタン クリックで確認ダイアログを表示し、確定後にすべての未読通知を既読化

3. 通知カテゴリタブ

コンポーネント 説明 機能・振る舞い
「すべて」タブ すべての通知を表示するタブ 件数バッジ付き。デフォルトで選択状態
「未読」タブ 未読の通知のみを表示するタブ 件数バッジ付き
「重要」タブ 重要度が高い通知のみを表示するタブ 件数バッジ付き
「タスク」タブ アクションが必要な通知のみを表示するタブ 件数バッジ付き
「システム」タブ システム関連の通知のみを表示するタブ 件数バッジ付き

4. 通知一覧セクション

コンポーネント 説明 機能・振る舞い
日付セパレータ 通知を日付ごとにグループ化するセパレータ 「今日」「昨日」「過去7日」などで表示
通知アイテム 個々の通知を表示する行（またはカード） クリックで通知詳細モーダルを表示。未読時は左側が青色でハイライト
通知アイコン 通知の種類を示すアイコン 種類に応じた色とアイコンを表示（重要:赤、タスク:橙、情報:青など）
通知タイトル 通知のタイトル 太字で表示。未読の場合は青い丸バッジを表示
通知内容プレビュー 通知の内容を簡略化して表示 2行程度で省略表示
通知日時 通知が発生した日時 相対時間（「1時間前」など）または日付で表示
アクションボタン 通知に対して実行できるアクションボタン マウスオーバー時に表示。既読/未読切替ボタンと削除ボタンを提供

5. ページネーションセクション

コンポーネント 説明 機能・振る舞い
表示件数選択 1ページあたりの表示件数を選択するドロップダウン 10/20/50/100件から選択可能。デフォルトは20件
ページナビゲーション ページ間を移動するためのページネーションコントロール 前後ページへの移動、特定ページへの直接移動が可能

6. モーダルダイアログ

コンポーネント 説明 機能・振る舞い
通知詳細モーダル 通知の詳細内容を表示するモーダルダイアログ 通知のタイトル、カテゴリ、時間、詳細内容、関連情報を表示。関連アクションへのボタンも提供
通知設定モーダル 通知設定を変更するためのモーダルダイアログ 通知タイプごとの受信設定（メール、アプリ内など）や保持期間の設定が可能

画面動作仕様

1. ページ初期表示時の動作

処理 内容
通知データ取得 ログインユーザー宛ての通知を新しい順に取得して表示
タブ選択 デフォルトで「すべて」タブが選択された状態
表示件数 1ページあたり20件の通知を表示
日付グループ化 通知を「今日」「昨日」「過去7日」などの期間でグループ化

2. タブ切り替え時の動作

操作 システム応答
「すべて」タブクリック すべての通知を表示
「未読」タブクリック 未読状態の通知のみを表示
「重要」タブクリック 重要度が高い通知のみを表示
「タスク」タブクリック アクションが必要な通知のみを表示
「システム」タブクリック システムからの通知のみを表示

3. 検索機能の動作

操作 システム応答
検索ボックスに文字入力 入力文字列を含む通知のみをリアルタイムでフィルタリング表示
検索条件に合致する通知がない場合 「該当する通知がありません」というメッセージを表示
検索ボックスの内容をクリア フィルタリングを解除し、元の通知一覧を表示

4. 期間フィルタリングの動作

操作 システム応答
期間選択ドロップダウンで項目選択 選択した期間内の通知のみを表示
「今日」選択 当日の通知のみを表示
「昨日」選択 前日の通知のみを表示
「過去7日」選択 過去7日間の通知を表示
「過去30日」選択 過去30日間の通知を表示
「カスタム期間...」選択 日付範囲選択用のポップアップを表示し、選択した範囲の通知を表示

5. 通知操作の動作

操作 システム応答
通知アイテムクリック 通知詳細モーダルを表示。未読の場合は自動的に既読に変更
既読/未読切替ボタンクリック 通知の既読/未読状態を切り替え、表示を更新
削除ボタンクリック 確認ダイアログを表示し、確定後に通知を削除（論理削除）
「すべて既読」ボタンクリック 確認ダイアログを表示し、確定後に表示されているすべての未読通知を既読に変更

6. 通知設定機能の動作

操作 システム応答
通知設定ボタンクリック 通知設定モーダルを表示
通知タイプごとの設定変更 各通知タイプの受信方法（アプリ内、メール）の有効/無効を切り替え
通知保持期間変更 通知の保持期間を選択（30日、60日、90日、180日）
設定保存ボタンクリック 変更した設定を保存し、モーダルを閉じる

7. レスポンシブ対応

画面サイズ 表示方法
デスクトップ・タブレット（768px以上） 標準レイアウト。通知を行形式で表示
モバイル（768px未満） カード形式に切り替え。各通知をカードとして縦に並べて表示

注意： 通知機能はユーザーにタイムリーな情報を提供するための重要な機能です。通知は最大90日間保持され、それ以降は自動的にアーカイブされます。アーカイブされた通知は「アーカイブ」タブから確認可能です。重要度の高い通知は視認性を高めるために赤色のアイコンで表示し、アクションが必要な通知には対応期限を表示します。通知詳細モーダルからは直接関連画面へ遷移できるようにし、ワークフローの効率化を図ります。また、リアルタイム通知機能を実装し、ページをリロードせずに新しい通知を表示します。未読の通知がある場合は、ヘッダー部分のアイコンにカウンターを表示して視認性を向上させます。
