勤怠・工数管理システム 画面設計

対象ユーザー

  • 技術者：日次の勤怠記録、工数入力を行うユーザー
  • プロジェクトマネージャー：チームメンバーの勤怠・工数を確認・承認するユーザー
  • 管理者：全社の勤怠・工数データを管理・集計するユーザー
  • 経理担当者：請求・支払業務のために工数データを参照するユーザー

業務フロー概要

技術者による勤怠入力 → 工数入力 → PMによる承認 → 集計・分析 → 請求・支払業務連携

勤怠・工数管理システムは、技術者の日々の勤務状況と案件ごとの作業工数を正確に記録・管理し、適切な承認フローを経て、請求・支払業務に必要なデータを提供します。また、集計・分析機能により、リソース配分の最適化やプロジェクト収益性の向上に貢献します。

📱

画面一覧

  • TMS-001 勤怠入力画面
    日次の勤務開始・終了時間、休憩時間、勤務場所などを入力する画面
  • TMS-002 工数入力画面
    案件・タスクごとの作業時間を記録する画面
  • TMS-003 勤怠管理画面
    管理者・PMが部下やチームメンバーの勤怠状況を確認・管理する画面
  • TMS-004 工数承認画面
    提出された工数データを確認・承認する画面
  • TMS-005 勤怠集計画面
    個人・部門・全社レベルでの勤怠データを集計・分析する画面
  • TMS-006 工数集計画面
    案件・プロジェクト・タスク種別などの軸で工数データを集計・分析する画面

実装上の注意点

  • 勤怠データと工数データの整合性を常に確保すること（工数の合計は実働時間を超えない）
  • カレンダーと連携し、祝日・休日を自動的に判定すること
  • 月次データの締め日・承認期限を設定可能にすること
  • 承認フローは組織構造やプロジェクト体制に合わせて柔軟に設定できること
  • 請求・支払システムとの連携を考慮したデータ出力機能を実装すること
  • モバイル対応により、外出先からの勤怠入力を可能にすること
