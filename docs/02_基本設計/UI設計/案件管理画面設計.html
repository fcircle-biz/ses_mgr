SES業務システム 案件管理画面設計

PRJ-001: 案件一覧画面

画面概要

項目 内容
画面ID PRJ-001
画面名 案件一覧
概要 全案件の一覧表示・検索・フィルタリングを行う画面
対象ユーザー 営業担当者、営業管理者、経営層
前画面 CMN-002: ダッシュボード
次画面 PRJ-003: 案件詳細、PRJ-002: 案件登録

画面レイアウト

[案件一覧画面のワイヤーフレーム]

画面構成要素

1. 検索・フィルター部分

項目ID 項目名 種類 説明
prj_list_001 検索ボックス テキスト入力 案件名、案件ID、顧客名などでのフリーワード検索
prj_list_002 案件ステータス セレクトボックス（複数選択可） 案件ステータスによるフィルタリング（検討中、提案中、進行中、完了、中止など）
prj_list_003 顧客 セレクトボックス 顧客企業によるフィルタリング
prj_list_004 担当者 セレクトボックス 営業担当者によるフィルタリング
prj_list_005 案件期間 日付範囲選択 案件の開始日・終了日によるフィルタリング
prj_list_006 スキル要件 マルチセレクト 必要スキルによるフィルタリング
prj_list_007 検索ボタン ボタン 検索条件を適用して結果を表示
prj_list_008 クリアボタン ボタン 検索条件をクリア

2. 一覧表示部分

項目ID 項目名 種類 説明
prj_list_009 案件一覧テーブル テーブル 検索条件に合致する案件の一覧
prj_list_010 案件ID テキスト（リンク） システム内部で管理される案件の一意識別子。クリックで詳細画面へ
prj_list_011 案件名 テキスト（リンク） 案件の名称。クリックで詳細画面へ
prj_list_012 顧客名 テキスト 案件の顧客企業名
prj_list_013 ステータス ラベル（色分け表示） 案件の現在のステータス
prj_list_014 開始日 日付 案件の開始予定日
prj_list_015 終了日 日付 案件の終了予定日
prj_list_016 担当者 テキスト 案件の営業担当者名
prj_list_017 アクションボタン ボタングループ 詳細表示、編集、マッチング検索などの操作ボタン
prj_list_018 ページネーション ページ制御 複数ページにわたる結果のナビゲーション

3. 操作ボタン

項目ID 項目名 種類 説明
prj_list_019 新規案件登録ボタン プライマリーボタン 新規案件登録画面への遷移
prj_list_020 CSVエクスポートボタン セカンダリーボタン 表示中の案件一覧をCSV形式でエクスポート

画面動作仕様

  1. 画面初期表示時は、デフォルトで「進行中」ステータスの案件を表示する
  2. 検索条件を指定して検索ボタンをクリックすると、条件に合致する案件が一覧表示される
  3. 一覧の案件ID・案件名をクリックすると、該当の案件詳細画面（PRJ-003）に遷移する
  4. アクションボタンの「詳細」をクリックすると、該当の案件詳細画面（PRJ-003）に遷移する
  5. アクションボタンの「編集」をクリックすると、該当の案件の編集モードで詳細画面に遷移する
  6. アクションボタンの「マッチング」をクリックすると、該当案件とのマッチング検索画面（MTH-001）に遷移する
  7. 「新規案件登録」ボタンをクリックすると、案件登録画面（PRJ-002）に遷移する
  8. 「CSVエクスポート」ボタンをクリックすると、現在の検索結果がCSVファイルとしてダウンロードされる
  9. ページネーションを使用して、複数ページにわたる検索結果をナビゲートできる

注意： ユーザーの権限レベルによって、表示される操作ボタンや実行可能な操作が異なります。

  • 営業担当者：自分が担当する案件のみ編集可能
  • 営業管理者：全案件の閲覧・編集が可能
  • 経営層：全案件の閲覧が可能だが、編集は不可

PRJ-002: 案件登録画面

画面概要

項目 内容
画面ID PRJ-002
画面名 案件登録
概要 新規案件情報を登録する画面
対象ユーザー 営業担当者、営業管理者
前画面 PRJ-001: 案件一覧
次画面 PRJ-003: 案件詳細（登録成功時）、PRJ-001: 案件一覧（キャンセル時）

prj_reg_019 キャンセルボタン ボタン - - 入力をキャンセルして前画面に戻る

画面動作仕様

  1. 必須項目が入力されているかをバリデーションチェックする
    • 未入力の必須項目がある場合はエラーメッセージを表示
    • 入力形式や値の範囲が不正な場合はエラーメッセージを表示
  2. 顧客セレクトボックスで「新規登録」を選択した場合、顧客登録用のモーダルダイアログを表示する
  3. スキル選択では、カテゴリーごとにスキルを選択できるツリー構造のセレクターを表示する
  4. 単価の下限が上限を超える値に設定された場合、エラーメッセージを表示する
  5. 「登録」ボタンをクリックした場合
    • 入力内容のバリデーションを行い、問題がなければ案件情報を登録する
    • 登録成功時は案件詳細画面（PRJ-003）に遷移する
    • 登録失敗時はエラーメッセージを表示し、入力内容を保持したままの状態で再表示する
  6. 「キャンセル」ボタンをクリックした場合
    • 入力内容を破棄して案件一覧画面（PRJ-001）に戻る
    • 入力内容がある場合は確認ダイアログを表示し、ユーザーの意思を確認する

PRJ-003: 案件詳細画面

画面概要

項目 内容
画面ID PRJ-003
画面名 案件詳細
概要 案件の詳細情報表示・編集を行う画面
対象ユーザー 営業担当者、営業管理者、経営層
前画面 PRJ-001: 案件一覧、PRJ-002: 案件登録
次画面 PRJ-001: 案件一覧、MTH-001: マッチング検索

画面レイアウト

[案件詳細画面のワイヤーフレーム]

画面モード

この画面は以下の2つのモードを持ちます：

  1. 表示モード: 案件の詳細情報を参照するためのモード（デフォルト）
  2. 編集モード: 案件の詳細情報を編集するためのモード（権限のあるユーザーのみ）

タブ構成

案件詳細画面は以下のタブで構成されます：

  1. 基本情報: 案件の基本的な情報（名称、期間、ステータスなど）
  2. 要件: 案件の詳細な要件（業務内容、スキル要件など）
  3. マッチング状況: 案件と技術者のマッチング状況
  4. 履歴: 案件に関する活動履歴

主要な機能

  • 案件の基本情報の表示・編集
  • 案件の詳細要件の表示・編集
  • 案件のステータス変更
  • 案件に関連するマッチング情報の表示
  • 案件の活動履歴の表示
  • マッチング検索の実行

画面動作仕様

  1. 画面初期表示時は、表示モードで基本情報タブが選択された状態で表示
  2. 各タブをクリックすると、対応する情報が表示されるようタブ切り替えが行われる
  3. 「編集」ボタンをクリックすると編集モードに切り替わる（権限のある場合のみ表示）
  4. 編集モードでの「保存」ボタンクリック時
    • 入力内容のバリデーションを行い、問題がなければ案件情報を更新する
    • 更新成功時は表示モードに戻り、更新された情報が表示される
    • 更新失敗時はエラーメッセージを表示し、入力内容を保持したままの状態で再表示する
  5. 「キャンセル」ボタンをクリックすると、編集内容を破棄して表示モードに戻る
  6. 「マッチング検索」ボタンをクリックすると、この案件の条件でマッチング検索画面（MTH-001）に遷移する
  7. 「一覧に戻る」ボタンをクリックすると、案件一覧画面（PRJ-001）に戻る

注意： 編集モードへの切り替えボタンは、編集権限を持つユーザー（案件の担当営業または営業管理者）にのみ表示されます。
